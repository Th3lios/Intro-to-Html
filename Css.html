CSS permite crear reglas que especifican como el contenido de un elemento deberia de aparecer. Por ejemplo se puede especificar que el fondo de una pagina puede ser de color rojo, o que todos los parrafos deberian de aparecer en gris usando arial 12, etc.

Css trabaja con ciertas reglas asociadas a elementos de HTML. Estas reglas se encargan de gobernar como el contenido debe mostrace. Una regla de CSS trata de 2 partes: selector y la delaration.

Ejemplo:
p(selector){
(declaration)font-family: Arial;
}

Como se puede ver el selector corresponde al elemento que se quiere modificar y la delaration la modificacion, en este caso se le cambio el estilo de letra a todos los elementos <p> de la pagina.

Dentro de la declaracion podemos ver que hay 2 tipos de datos, la propiedad (font-family) y el valor (Arial). Tu puedes declarar muchas propiedades dentro de una declaracion. Tambien puedes colocar varios selectores con una sola declaracion, por ejemplo:

Ejemplo:
p, h1, h2, h3{
font-family: Arial;
}

COLOR

Color de primer plano

La propiedad color permite especificar el color del texto dentro del elemento. Puedes especificar cualquier color en una de las 3 siguientes maneras:

  • rgb values: esta expresion permite colorear en termino de que tanto rojo, verde y azul es utilizado para colorear. Por ejemplo: color: rgb(100,100,90);
  • hex code: esta expresion de 6 digitos que es representado en un monto de rojo, verde y azul, precedido por un simbolo #. Por ejemplo: color: #ee3e80;
  • color name: Hay 147 nombres de colores predefinidos que pueden ser reconocidos por un browser. Por ejemplo: color: DarkCyan;

Color de fondo

Se habia hablado previamente que habia que imaginarse un cuadrado imaginario al rededor de cualquier elemento de HTML, bueno bajo ese cuadrado trabaja la propiedad background-color. Se puede especificar el color del fondo de las mismas 3 maneras previas color name, rgb(,,,) o hex code. Si no se especifica el color del fondo, este por default sera transparente. Por default algunos browsers aplican sus propias especificaciones, entonces para asegurarte de que sea blanco, puedes especificar el color blanco en el fondo.

#header3{

Todos los colores en la pantalla del computador estan compuestos por la combinacion del rojo, verde y azul. Para encontrar el color que tu quieres deberias de usar un mezclador, como el de photoshop o alguna pagina en internet.

HUE
Hue es cercano a la idea coloquial de color. Tecnicamente hablando, sin embargo, un color puede tener saturacion y luminosidad.
Saturation
Saturacion se refiere a la cantidad de gris en un color. El maximo de saturacion deberia de no ser el gris en un color, en cambio el minimo deberia de estar muy cercano al gris.
Brightness
El brillo se refiere a que tanto negro hay en un color. El maximo de luminosidad deberia ser muy lejano al negro, en cambio con poco brillo deberia de ser muy cercano al negro.

Contraste

Cuando elegimos un color de primer plano y de fondo, es importante asegurarse de que tiene el suficiente contraste para que el texto sea legible.

Contraste bajo, el texto es dificil de leer cuando es bajo el contraste entre el fondo y las letras

Contraste alto, el texto es facil de leer cuando hay alto contraste entre el fondo y las letras.

Contraste medio, si el texto tiene un fondo oscuro y letras claras, puedes incrementar la separacion entre las lineas y el peso de la fuente para hacer mas facil la lectura.

CSS3: opacity

Css3 introduce la propiedad de que permite opacar el color de los elementos. El valor es el numero entre 0.0 y 1.0 (El valor 0.5 equivale al 50% de opacidad, etc). Este cambio con la propiedad opacity afecta a los hijos. (red, opacity 0.5)

El rgba permite especificar el color, pero ademas se le anadio un cuarto valor que permite cambiar la opacidad de color, al igual como se explico antes, este valor va de 0.0 a 1.0 cumpliendo las mismas caracteristicas. El cambio de valores en rgba solo hace cambios en donde se aplica, no cambia a los hijos. (rgba 0,0,0,0.5)

Hay otra propiedad introducida en CSS3 llamada hsl, que utiliza hue, saturacion y luminosidad, donde el hue se representa desde 0 a 360, la saturacion de 0% a 100% y la luminosidad tambien de 0% a 100%. Notar que la luminosidad es un concepto distinto al brillo. El brillo quita o anade negro, en cambio la luminosidad anade blanco (0%) y negro(100%) (normal 50%) para anadir o quitar.

body{
background-color: #C8C8C8;
background-color: hsl(0,0%,78%);}
hsl{
background-color: #ffffff;
background-color: hsla(0,100%,100%,0.5);
}

Como se pudo ver en el ejemplo se utilizo un nuevo atributo, ya no es el hsl, sino hsla el cual anade un elemento mas ademas de HUE, saturacion y luminosidad. hsla anade el alpha, esta expresion es un numero entre 0.0 y 1.0, el cual representa la transparencia de un color.

Border, Margin y Padding

Cada caja tiene 3 propiedades disponibles para ajustar su apariencia.

border
Cada caja tiene un borde (incluso si esta no es visible). El borde separa la orilla de una caja con otra.
margin
El margen corresponde a la orilla que va afuera del borde. Puedes configurar el ancho del margen para crear una brecha entre los bordes de 2 cajas.
padding
El relleno corresponde al espacio entre el borde y cualquier contenido en la caja, Anadiendo relleno puedes incrementar la redaccion del contenido.

La propiedad border-width controla el ancho de un borde. El valor de esta propiedad puede ser dado en pixeles, porcentaje, ems o usando uno de estos valores: thin,medium o thick.

Se puede controlar de manera individual los lados del borde con border-top-width, border-right-width, border-left-width o border-bottom-width.

Tambien se puede especificar los diferentes width asi: border-width: 2px 1px 1px 2px;

Se puede controlar el estilo de los bordes que se pueden utilizar con la propiedad border-style, la cual puede tomar los siguientes valores:

solid
Una unica linea.
dotted
Una serie de punos cuadrados (si el borde es de 2px de ancho, entonces los puntos son 2px cuadrados con 2px de brecha entre cada punto).
dashed
Una serie de lineas cortas
double
Dos solidas lineas (el valor del border-width crea una suma de las 2 lineas
groove
Aparece una linea tallada en la pagina
ridge
Parece como si los bordes estuviesen saliendo de la pagina
inset
Parece como si estuviera incrustado en la pagina
outset
Parece como si el relleno estuviera saliendo de la pagina
hidden/none
No hay bordes

Tambien se puede modificar un color de borde en especifico con: border-(left,right,top y bottom)-color:;

Controlando la posicion de los elementos

CSS tiene un esquema de posicionamiento que permite el control del diseno de la pagina: Flujo normal, posicion relativa y posicion absoluta. Especifica la posicion del esquema utilizado con la propiedad position. Puedes tambien dejar flotando un elemento con la propiedad float.

  • Normal flow: Cada elemento de bloque aparece en una nueva linea, causando que cada objeto aparezca abajo de la pagina mas que el anterior. Incluso si especificas el ancho de la caja y su espacio entre 2 elementos situados uno al lado de otro, ellos no apareseran juntos. Este es el comportamiento por default.
  • Relative positioning: Mueve un elemento desde la posicion donde deberia de estar normalmente, llevandolo a la cima, a la derecha, abajo o a la izquierda de donde deberia de estar. Este no afecta la posicion de los demas elementos alrededor. Ellos estan en una posicion normal.
  • Absolute positioning: Este elemento esta en relacion con su contenido. Lo saca de su posicion normal, lo que no modifica la posicion de los demas elementos a su alrededor.

Para indicar donde una caja debe ser posicionada, necesitas usar la propiedad box offset para decirle al browser que tan arriba, abajo, derecha o izquierda deberia de estar posicionado.

Fixed position: esta es una forma de una posicion absoluta que posiciona un elemento en una relacion con la ventana, como oponiendoce al elemento contenido.

Floating elements: Un elemento de flujo permite mover ese elemento fuera del flujo normal y posicionarlo a la izquierda o a la decrecha del contenido de la caja.

En flujo normal, cada elemento de bloque situado encima del siguiente. Este es el camino por default el cual los browsers usan para los elementos de HTML, No se necesita CSS para indicar que elemento debe aparecer en un flujo normal, pero el syntaxis puede ser:

position: static;

La posicion relativa mueve un elemento en relacion a donde este a estado en su flujo normal. Por ejemplo mover un elemento 10pixeles desde su posicion normal hacia abajo. Puedes indicar que un elemento debe ser posicionado relativamente con el valor position: relative;

Cuando la propiedad position toma el valor absolute, la caja es tomada fuera de su posicion normal. La caja sin configuracion especifica donde el elemento deberia de aparecer en relacion a su elemento contenido.

Fixed position es un tipo de posicion absoluta que requiere la propiedad position con el valor fixed

    Por mucho tiempo, los creadores de paginas web han usado el elemento div para agrupar todo los elementos. Autores han usado clases o id's para indicar el rol del div en la pagina.

    Se puede ver el diseno comunmente usado en blogs.

    • La parte superior de la pagina es el encabezado, el cual contiene el logo y la navegacion primaria de la pagina.
      navegacion primaria: corresponde a las secciones permitidas en la pagina como CONTACTO, HOME, QUE HACEMOS, etc.
    • Abajo de esto esta uno o mas articulos del blog (como post, etc)
    • En el lado derecho de la zona del post del blog, a veces hay topicos de otros post del mismo blog
    • Cuando codificamos un sitio web como este, los desarrolladores pueden usualmente poner la parte principal de la pagina bajo un div y usar los atributos class o div para indicar su proposito.

    Nuevo elementos de diseno en HTML5

    HTML5 introduce un nuevo set de elementos que permiten dividir las partes de la pagina.

    Este ejemplo exactamente tiene la misma estructura que la pagina previa. Sin embargo, muchos de los div han sido reemplazados por el nuevo diseno del HTML5

    Por ejemplo, el encabezado situado dentro de un nuevo elemento, header. Un elemento de navegacion, nav. El articulo esta en un elemento individual llamado article.

    El punto de creacion de estos nuevos elementos es para que el autor de la web pueda usarlos en ayuda para describir la estructura de la pagina. Por ejemplo, un software de lectura puede permitir a los usuarios ignorar el encabezado y el pie de pagina, y solo leer el contenido (content). Similarmente, los buscadores pueden soportar mas peso en el contenido del elemento article que en el header o footer. Esto hace el codigo mas facil de leer.

    Los elementos header y footer pueden ser usados para:

    • El contenido principal del encabezado o el pie de pagina puede aparecer en la cima o al final de cada pagina de un sitio
    • Un encabezado o pie de pagina para un article o section en cada pagina

    En el ejemplo, el header es usado para contener el nombre del sitio y el contenido principal de la navegacion. El footer contiene el contenido copyright, junto con links de terminos y condiciones de privacidad.

    Cada elemento article y section pueden tener su propio header y footer para mantener la informacion de header o footer en la seccion indicada en la pagina. Por ejemplo, en una pagina con muchos post, cada post por individual puede ser pensado con diferentes secciones. El header puede ser utilizado para contener el titulo y la fecha de cada post. El footer puede contener el enlace para compartir el articulo en un social networking site.

    El elemento nav es usado para contener los bloques con mayor contenido de navegacion en un sitio como la navegacion principal (home,classes,gatering,about,contact).

    Volviendo a lo del blog, por ejemplo, si quieres terminar un articulo con un link para relacionarlo con otros post, este puede no contar como un bloque de mayor navegacion y por eso no deberia de ser situado dentro de un nav

    Al mismo tiempo algunos de desarrolladores que ya usan HTML5 deciden usar el elemento nav para los enlaces que aparecen en el final de cada pagina (links para cosas de terminos de privacidad, etc).

    El elemento article actua como un contenedor de cualquier seccion de una pagina que puede actuar sola y potencialmente ser organizado.

    Este puede ser un articulo individual o entrada de algun blog, un comentario, post de un foro, o muchas otras piezas de contenido individual.

    Proceso y diseno

    Esta seccion discute un proceso que puede ser usado cuando estas creando un nuevo sitio web.

    Esto va hacia quienes estan visitando el sitio y como asegurarce que las caracteristicas de la paginas son las que los visitantes necesitan.

    • Como entender que atrae a la audiencia de tu sitio y que informacion ellos esperan encontrar en el.
    • Como organizar la informacion de manera que los visitantes puedan encontrar lo que estan buscando.
    • Teoria del diseno para presentacion de la informacion de manera que ayude a los visitantes alcanzar sus metas.
    • Tips de diseno para ayudar a hacer mas atractivo y profesional tu sitio.

    Para quienes va dirigido el sitio.

    Cada sitio deberia de ser disenado para un tipo de audiencia en especifico, No solo para ti mismo o para el dueno del sitio. Por eso es muy importante entender para que audiencia va dirigido.

    Puede ser de ayuda hacer algunas preguntas sobre que es lo que esperan y seria interesante en el tema de tu sitio.

    Si le preguntas al cliente para quien va digido el sitio, no es poco comun que te digan que "para todo el mundo".

    Siendo realistas, es improbable que sea aplicable para todos. Si tu sitio vende apolletas, aunque la mayoria de las personas que usan computadores probablemente usen ampolletas, es poco probable que las pidan de otro pais.

    Incluso si el sitio tiene diferentes atracciones, puedes seguir pensando sobre un publico en especifico.

    Consiguiendo audiencia: Individual

    • Para que rango de edad va dirigido
    • Para que tipo de sexo va dirigido o si es mixto.
    • En que pais viven los visitantes
    • Si son de zonas rurales o urbanas
    • Promedio de ingresos de los visitantes
    • Nivel de educacion de los visitantes
    • Status marital o familiar
    • Que ocupacion tienen
    • Cuantas horas trabajan por semana
    • Que tan a menudo usan la web
    • Que tipo de dispositivo usan para acceder a la web

    Consiguiedo audiencia: Companias

    • De que tamano es la compania o su departamento mas relevante
    • Cual es la posicion de las personas en la compania quienes visitan su pagina
    • Si visitantes usaran la pagina para si mismos o para alguien mas
    • Cual es el presupuesto que manejan

    Inventa un visitante ficticio de tu audiencia. Pueden ser tus amigos. Ellos pueden influir en el desarrollo del diseno del color o detalles en la descripcion.

    Por que la personas visitan tu sitio web.

    Ahora que se conoce el tipo de visitante, tiene que considedar el "por que" de que vengan a tu sitio. Mientras algunas personan simplemente pasan de largo.

    Tu contenido o diseno debe influir en el timpo de meta de los usuarios.

    Para ayudar a determinar porque la gente viene a tu sitio, hay 2 categorias basicas de preguntas que puedes hacer.

    • Lo primero es intentar descrubir la fuente de motivacion por el cual los visitantes visitan la pagina.
    • Lo segundo consiste en examinar la meta especifica de los visitantes. Hay muchos gatilladores que hacen que la visiten

    Motivaciones

    • Estan buscando por lo general entretenimiento o necesitan alcanzar una meta en especifico?
    • Si tiene una meta en especifico, es personal o prefesional?
    • Gastan tiempo en actividades esenciales o en lujos?

    Metas especificas

    • Hacer lo que ellos quieren en la informacion general / investigar (como el background en un topico), o hay algo especifico (como un hecho en particular o informacion de un producto).
    • Estan familiarizados con el servicio o producto que ofrecen o estan necesitando ser introducidos a este?
    • Estan buscando tiempo para informacion sensible, tales como las ultimas noticias o actualizaciones en un topico en particular?
    • Quieren descubrir informacion sobre un producto en especifico para ayudar a decidir si comprarlo o no?
    • Necesitan contactarte? Si es asi, pueden visitarlos en persona? o pueden necesitar un email o telefonos de contacto?

    Que visitantes estan intentando de alcanzar

    Es dificil que tu puedas anotar cada razon del porque visitan tu sitio pero tienes que buscar claves para su motivacion. Es informacion que puede ayudar a guiar tu sitio.

    Lo primero que hay que hacer es crear una lista de razones porque las personas pueden visitar tu sitio. Puedes asigan la lista de tareas a un visitante ficticio como se explico antes.

    Ejemplo:

    • Gordon compro unos tenis hace unos anos: hora quiere comprar unos desde tu sitio para su novia
    • Jasper tuvo una mala experiencia en un hotel cuando visitaba Sydney, Australia, y quiere hacer una queja.
    • Ayo espera estudiar arquitectura para querer aprender mas sobre un nuevo curso que se esta ofreciendo.

    Que informacion necesitas

    Conoces quienes estan comenzado a visitar tu sitio y porque lo hacen, ahora necesitas trabajar sobre que informacion necesitan en orden para alcanzar sus metar rapida y efectivamente.

    Quizas quieres ofrecer soporte adicional de informacion que crees que puedes ser de ayuda.

    Tienes que revisar cada razon por la cual la gente visita tu sitio y determinar cuales son las metas que quieren alcanzar

    Por eso, tendras mas oportunidades para decirles cualquier informacion extra que crees de ayuda (o exponerlas para otros productos y servicios que ellos quieran mejorar).

    Algunas preguntas que te ayudaran a decidir que informacion proveer para los visitantes del sitio.

    • Los visitantes estaran familiarizados con tu tema/ marca o necesitas introducirlo tu mismo?
    • Estaran familiarizados con el producto/servicio/informacion que cubres o necesitaran informacion en el background del sitio?
    • Que es especial sobre que ofrecer que te diferencia de otros sitios?
    • Que caracteristicas son mas importantes de lo que ofreces?
    • Una las vez personas hayan alcanzado sus metas que enviaron a tu sitio, quedan preguntas que hacer sobre este tema?

    Que tan a menudo las personas visitaran tu sitio

    Algunos beneficios de los sitios es comenzar a actualizarse mas seguidos que otros. La informacion puede cambiar constantemente, mientras que otro contenido puede mantenerce relativamente estatico.

    Un website de moda va a cambiar mas seguido que uno que ofrece un servicio a personas el cual no es comprado tan regularmente.

    Una vez la pagina a sido construida, puede tomar mucho tiempo y recursos actualizarla frecuentemente.

    Algunas preguntas que ayudan a decidir que tan a menudo actualizar

    Buenos/servicios

    • Que tan a menudo las personas volveran a comprar?
    • Que tan a menudo tu stock sera actualizado o que tus servicios cambien?

    Informacion

    • Que tan a menudo los temas son actualizados
    • Que porcentaje de tus visitantes vuelven regularmente al tema, comparado con quienes solo necesitan informacion una vez.

    Ahora que conoces que necesitas que aparezca en tu sitio, tienes que comenzar a organizar la infomacion en secciones o paginas.

    Lo principal es crear un diagrama de la pagina que tu usaras en la estructura de tu sitio. Esta es conocida como site map y esto sera mostrado como paginas que pueden agruparse.

    Para ayudar a decidir que informacion deberia ir en cada pagina. Puedes usar la tecnica llamada card sorting.

    Esto implica colocar cada pieza de informacion que un visitante puede necesitar saber, en piezas separadas de papel y entonces organizar la informacion relacionada en grupos.

    Cada grupo relacionado es relacionado a una pagina y en largos sitios, las paginas pueden volverse a agrupar para crear diferentes secciones de un website.

    Un site map generalmente comienza con el homepage. Adicionalmente, si el sitio es largo y es compartimentado en secciones, cada seccion puede requerir su propia seccion de homepage para enlazar toda la informacion en el.

    Por ejemplo, la mayoria de tiendas online tiene la seccion homepage para cada tipo de producto, Que a su vez enlazan a paginas de productos individuales.

    Wireframes

    Un wireframe es un simple sketch de la informacion clave que necesitas para colocar en cada pagina de un sitio. La informacion es mostrada de manera jerarquica y segun el espacio que requiera

    Muchos disenadores toman elementos que nesecitaran que aparezcan en cada pagina y comenzaran por la creacion del wireframe. Esto incluye sketching o sombreado de areas donde cada elemento de la pagina ira.

    Creando un wireframe tu puedes asegurar que todo la informacion que necesitas va a estar en una pagina.

    No deberias de incluir colores, fuentes, background o imagenes para el website en el wireframe. Este deberia de enfocarse en que informacion tiene que estar en cada pagina y crear una jerarquia visual para indicar que es mas importante en cada pagina

    El wireframe hace el diseno mas facil porque puedes saber que informacion necesita aparecer en cada pagina antes de considerar como la pagina deberia verse. Puede ser de ayuda mostrar el wireframe del sitio al cliente antes de mostrarle el diseno. Esto le permite al cliente asegurarse del que el sitio cumple todas las funciones y tiene la informacion que requiere ofrecer.

    Si solo presentas el diseno del sitio al cliente, es comun para rellos enfocarse solo en como se ve, lo que significa que es posible que ellos no mejoren problemas de funcionalidad despues de que el sitio sea construido.

    Agrupando y Similaridad

    Cuando hacemos tomar sentido a un diseno, tendemos a organizar elementos visuales en grupos. Agrupando piezas de informacion relacionadas, puede hacer un diseno facil de comprender. Aqui hay algunos ejemplos de como agruparlos:

    Proximidad: Cuando muchos objetos estan posicionados juntos, se perciben como una relacion a diferencia de objetos que estan colocados muy apartados.

    Acercamiento: Cuando te enfrentas con una complicada estructura de objetos, a menudo vamos en busca de un unico o reconocible patron o forma. Una caja imaginaria o real puede ser formada alrededor de elemento debido a su proximidad y alineamiento.

    Continuidad: Cuando elementos estan posicionados en una linea o curva, entonces son percibidos para ser relacionados mas que con los que no siguen la misma direccion. Esto puede ser usado para direccionar al lector desde una parte de una pagina a la siguiente.

    Espacios en blanco: Posicionando objetos relacionados cerca y dejando una gran brecha entre objetos no relacionados.

    Color: Un fondo con un color detras de objetos relacionados para enfatizar su conexion.

    Bordes: Una linea puede ser dibujada alrededor del borde de un grupo o entre estos y sus vecinos.

    Naturalmente observamos similaridades en los disenos, y cosas como esas son percibidas para ser mas relacionadas con cosas que son poco similar. Repeticion de colores, tamanos, texturas, fuentes o formas provoca una comparacion de elementos que tiene importacia o significado similar.

    Disenando el area de navegacion

    La navegacion del sitio no solo ayuda a las personas a encontrar donde quiere ir, tambien les ayuda a entender de que trata tu sitio y como esta organizado. Una buena navegacion tiene a seguir los siguientes principios.

    Consistencia

    Idealmente, la navegacion deberia de ser rapida y facil de leer. Es buena idea intentar de limitar el numero a no mas de 8 opciones. Los opciones pueden cambiar si estas en la homepage o en otra pagina.

    Limpieza

    Los usuarios deberian de saber predecir el tipo de informacion que pueden encontrar en la pagina antes de clickear en el link. Donde es posible elegir una sola palabra descriptiva para cada link en vez de frases.

    Selectividad

    La navegacion principal deberia solo reflejar las secciones o contenido del sitio. Funciones como login o search, y informacion legal como terminos y condiciones son ubicados mejor en otro lado.

    Contexto

    Una buena navegacion tiene contexto. Deja a los usuarios saber donde estan en la pagina en ese momento. Usando diferentes colores o algun tipo de marca visual para indicar la pagina actual es una buena idea.

    Interactividad

    Cada link deberia de ser lo suficientemente grande para ser clickeado y la apariencia del link deberia de cambiar cuando el usuario tiene el cursor sobre el y cuando es clickeado. Tambien deberia de ser distinto de otro contenido de la pagina.

    Consistencia

    La mayoria de las paginas que tiene un gran numero de objetos de navegacion, tienen un segundo seccion de navegacion que cambia de pagina en pagina. Es la mejor manera de mantener la navegacion primaria exactamente igual.

    Web Hosting

    Para que otra persona pueda ver tu sitio, tendras que subirlo a un servidor web. Los servidores web son computadores especiales que estan constantemente conectados a internet. Son especiales para mantener paginas web cuando se les solicita.

    Con la excepcion de algunos sitios muy grandes, la mayoria de los sitios web estan en servidores web de companias de host. Normalmente son mas baratos y mas confiables que tratar de correr tu propio servidor web

    Hay muchos tipos de oferta de hosting. Se van a ver algunos tips para saber cual compania de hosting contratar.

    • Disk Space: Se refiere al tamano total de los archivos adornan tu sitio (Todo el HTML y CSS entre imagenes y scripts)
    • BandWidth: Es la cantidad de data que la compania de hosting enviara a tu sitio de visitantes. Si imaginas a 10 personas mirando en cada pagina de tu sitio, entonces este equivale a 10 veces la cantidad de disk space en tu sitio.
    • BackUps: Checkea si la compania de hosting realiza backups en tu sitio y que tan a menudo. A veces solo crean backups para que ellos puedan restaurar tu sitio web en caso de que el server caiga. Otros permiten acceder al backups (Lo cual puede ser beneficioso si tu accidentalmente rompes tu sitio en alguna actualizacion)
    • Email Accounts: La mayoria de las companias de hosting proveeran un email server con su propio paquete de hosting. Queras checkear el tamano del mailbox cuando se te permita y tambien el numero de mailboxes que puedes usar.
    • Server-side languages and databases: Si estas usando un sistema de manejo de contenido, sera util itilizar un server-side lenguaje de programacion (como php con mysql o ASP.NET con sql server). Asegurate de checkear que tu compania de host soporte esas tecnologias.

    a menudo uno se deja llevar por la calidad de los hosting por los comentarios. Desafortunadamente solo podras saber eso cuando algun error ocurre o hasta que punto estan dispuesto a ayudarte (asi que puedes esperar a ver un poco de negatividad en comentarios de algunas companias)

    CSS Animacion

    Hay una gran variedad de elementos que pueden ser animados, pero no todos deberian de serlo

    En CSS hay 2 formas principales para darle movimiento a los elementos

    • Transition property
    • Animation property + keyframes

    Transition property

    La sintaxis de la propiedad transition esta compuesta de 4 valores:

    .element{
    transition: [property] [duration] [ease] [delay] }

    Ejemplo: .element{
    transition: opacity 300ms ease-in-out 1s }
    El delay corresponde el tiempo de espera para que se produzca la transicion.

    El valor ease puede ser:

    • ease: Este es el valor por default. Especifica un efecto de transicion con un comienzo lento, luego rapido y despues con un final lento (equivalente a cubic-bezier(0.25,0.1,0.25,1))
    • linear: Especifica un efecto de transicion con la misma velocidad desde el inicio hasta al final (equivalente a cubic-bezier(0,0,1,1))
    • ease-in: Especifica un efecto de transicion con un comienzo lento (equivalente a cubic-bezier(0.42,0,1,1))
    • ease-out: Especifica un efecto de transicion con un termino lento (equivalente a cubic-bezier(0,0,0.58,1))
    • ease-in-out: Especifica un efecto de transicion con un comienzo y un final lento (equivalente a cubic-bezier(0.42,0,0.58,1))
    • step-start: Equivalente a steps(1, start)
    • step-end: Equivalente a steps(1, end)
    • steps(int,start|end): Especifica una funcion step, con 2 parametros. El primer parametro especifica el numero de intervalos en la funcion. Tiene que ser positivo mayor a 0. El segundo parametro, con opciones de start o end, y especifica el punto en el cual el cambio de valor ocurre en el intervalo. Si el segunto parametro es omitido, es se le da el valor end.
    • cubic-beizer(n,n,n,n):Define tu propio valor en el la funcion cubic-beizer. Posibles valores son numericos y van de 0 a 1.
    • initial:Configura esta propiedad con este valor por default.
    • inheritInherits esta propiedad de si elemento padre.

    Keyframes

    Los keyframes son faciles de entender, esta es la sintaxis

    @keyframes [name]{
    from{
    [style];
    }
    to{
    [style];
    }
    }

    El nombre se usara para llamar al selector, los keyframes no van dentro de los selectores del todo.

    Dentro del keyframe hay 2 parametros el from y el to y dentro de ellos esta style que va a ser lo que va a cambiar ejemplo:

    @keyframes myframes{
    from{
    height:200px;
    background:orange;
    }
    to{
    height:200px;
    background:pink;
    }
    }

    En vez de from y to, puedes cambiarlo por porcentajes el cual representa el porcentaje de tiempo que ha pasado por ejemplo 0s a 2s es igual a 0% a 100%

    @keyframes myframes{
    0%{
    height:200px;
    background:orange;
    }
    100%{
    height:200px;
    background:pink;
    }
    }

    La propiedad animation tiene una gran variedad de valores que puede tomar, esta es su sintaxis:

    .element{
    animation: [name] [duration] [timing-function] [delay] [iteration-count] [direction] [fill-mode] [play-state]}

    ejemplo:
    .element{
    animation: myframes 2s ease-in-out 0s infinite normal forwards paused}

    • name: El nombre del frame
    • duration: tiempo que quieras que se demore la animacion en completarse (2s)
    • timing-function: de que manera se demorara mas en el inicio, en el final, en el medio (ease)
    • delay: tiempo que se demorara en comenzar la animacion por primera vez (5s)
    • iteration-count: las veces que se repetira la animacion (5) (infinite)
    • direction: como va a hacerce la animacion (normal) o (reverse) o (alternate nomal y reverse)
    • fill-mode: Controla como el elemento se ve despues de la animacion como (forward) que significa que termina en como se ve al final de la animacion o (normal)
    • play-state: puede ser (running) o (paused)

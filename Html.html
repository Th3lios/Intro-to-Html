Elementos de formas

Todo elemento de forma comienza con el elemento (form), donde siempre deberia de contener al atributo (action), el cual contiene la url de la pagina de la cual el servidor recibe la informacion cuando esta es entregada, tambien existe el atributo (method), que puede contener 2 valores, get o post.

El get method, es anadido despues de haber tipificado ya el atributo action="url"

El get method, es ideal para:
-formas cortas, tales como las cajas de busqueda (search boxes) -Cuando estas recuperando data del servidor (No enviando informacion que deberia ser anadida o eliminada de la base de datos)

Con el post method, los valores son enviados en lo que se conoce como HTTP header. Se deberia de usar el post method, si la forma:
-Permite a los usuarios enviar archivos -Es muy largo -Contiene datos sensibles (contrasenas) -Anade o elimina informacion de una base de datos

Si el atributo method, no es anadido quedara por default como method="get"

Hay un atributo en casi todos los elementos el cual es id, que sirve para darle un valor de identificacion a un elemento en especifico, por ejemplo: id="formas"

Existe tambien el atributo size el cual no deberia ser utilizado en las nuevas formas, pero en las antiguas se utilizaba para darle un ancho al texto de entrada (el tamano de la search box por ejemplo)

Por otra parte esta el elemento (input) un elemento vacio, que va dentro del elemento (form), que es usado para crear varias formas de control, y el valor del atributo type que va en el elemento (input), va a indicar que tipo de forma va a ser, por ejemplo: (p)Username:(input type="text" name="username" size="15" maxlength="30" /)(p) (forma tipo text, nombre de la forma username, tamanho de la forma 15 y maximo de caracteres 30)

En HTML existen los elementos de bloque y los elementos de linea. Los elementos de bloque son todos aquellos elementos que encapsulan lineas completas, por ejemplo ul,p,h1,etc. En cambio los elementos de linea son aquellos elementos que encapsulan texto dentro de las lineas, por ejemplo: a,b,em,img,etc

Existe un elemento importante en HTML que se llama div-/div, el cual ayuda a encapsular en bloque un set de elementos. Se utiliza principalmente para estilizar con CSS y de ordenar

Hay otro elemento parecido al div que se llama span, el cual funciona igual que el div pero como un elemento de linea. La razon principar de usar span en un codigo, es que se quiera controlar la apariencia de un texto en especifico.

El elemento (meta) vacio, vive dentro de el tag head y contiene informacion sobre la pagina web. No es visible para el usuario pero tiene un numero de propositos como decir a los motores de busqueda, acerca de la pagina, quien la creo y si tiene un tiempo de vida. El atributo de name que sele asigna al meta se encarga de indicar la intencion que va a tener la descripcion especificada. El atributo content, es donde la especificacion es descrita.

Los diferentes valores que toma comunmente el atributo name en el meta por lo general son:

description
Contiene la descripcion de la pagina. Esta descripcion es comunmente usada por los motores de busqueda para entender que tipo de pagina y debe tener un maximo de 155 caracteres.
keywords
Contiene separados por coma las palabas que el usuario podria buscar al encontrarlas en la pagina
robots
Esto indica si el motor de busqueda deberia de anadir esta pagina a sus resultados de busqueda o no. Un valor de nofollow puede ser usado si el motor de busqueda deberia anadir esta pagina en sus resultados pero no cualquier pagina que la enlaza a ella.

El elemento meta, tambien usa el atributo http-equiv y el atributo content simultaneamente. http-equiv, puede ser author que define al autor de la pagina, ejemplo: (meta http-equiv="author" content="Jon Duckett" />), pragma que evita que el browser almacene la pagina (consiste en almacenar cache localmente para ahorrar tiempo cuando se vuelva a antrar) ejemplo : (meta http-equiv="pragma" content="no-cache" />) y expires que debido a que los browsers amenudo almacenan el contenido de la pagina, la opcion expires puede ser usada para indicar cuando la pagina deberia de expirar y avisano al browser cuando deberia de dejar de almacenar cache de la pagina y debe ser especificado de esta manera (meta http-equiv="expires" content="Fri, 04 Apr 2014 23:59:59 GMT" />)

El camino mas popular para anadir un video flash en la pagina web es usando javaScript, hay muchos codigos dispuesto a hacer esto sin necesidad de saber de manera mas profunda como funciona el codigo. El script que se esta buscando se llama SWFObjetc, del cual se pude obtener una copia gratis de google la cual puede ser usada libremente.

La tecnica consiste en utilizar div para crear un espacio donde el flash video deberia de ubicarse. El div tiene una id del cual su valor es usado por SWFObject script para desplegar el video en el div, en este caso la id del div es id="bird" y este es el script en el head: swfobject.embedSWF("flash/bird.swf","bird", "400", "300", "8.0.0"); . Dentro del div puedes poner un comentario para aquellos que no pueden reproducir videos flash. (recordar que el script va en el head)

A pesar de que HTML5 <video> a sido recientemente anadido, tiene grandes capacidades de uso:

Soporte
Este nuevo elemento de HTML5 solo tiene soporte por los nuevos browsers, asi que no se puede usar esta tecnica si se quiere mostrar el video a todo usuario
Digital rights
Igual que la escritura del elemento video no tiene soporte por muchos tipos de Digital rights management (Manejo de derechos digitales). Pero un pirata dedicado usualmente encontrara una manera de rodear el DRM
Formatos
No todos los browsers soportan todos los formatos de video. Por eso necesitas proveer tu video en diferentes formatos. Para alcanzar la mayoria de browsers deberias de subir el video en los siguientes formatos: H264: IE and Safari SWebM: Android, Chrome,Firefox, Opera
Controles

Uno de los problemas con los reproductores tales como flashplayer es que ellos pueden comportarce de manera inconsistente cuando elementos como menus se despliegan, o la ventana es escalada de arriba a abajo. Las opciones de HTML5 arreglan este problema.

HTML5: Anadiendo videos a tu pagina

<video> tiene un numero de atributos que permite el control de video (playback). Como ya vimos video al igual que otros elementos tiene un elementro llamado src, el cual recibe la ubicacion del video.

El atributo poster permite colocar una imagen de portada al video antes de la reproduccion.

Tambien tiene los atributos width y height.

El atributo preload le dice al browser que hacer cuando la pagina carga, este puede tener 1 de 3 valores:

none que le dice al browser que no debe de cargar el video hasta que el usuario aprete play.
auto que le dice al browser que debe de descargar el video cuando la pagina carga.
metadata el browser deberia de recolectar informacion como el tamano, first frame, la lista de videos (track list) y duracion.

El atributo controls cuando es usada la ventana, este atributo indica que el browser deberia de aportar su propio control de playback

El atributo autoplay cuando es usada la ventana, este atributo especifica que el archivo deberia de reproducirce automaticamente

el atributo loop cuando es usada la ventana, este atributo indica que el video deberia de empezar a reproducirce una vez alla terminado

Multiple video source. Dentro del elemento video, existe otro elemento llamado source, utilizado para especificar la localizacion del archivo a reproducir, este deberia de reemplazar el atributo src de video. Puedes usar multiples source para especificar que el video esta disponible en varios formatos.(debido a bugs de ipad el formato mp4 deberia de ser el primero en mostrase, de otra manera no se reproduciria.

Dentro de source hay atributos como src para la ubicacion del archivo, type para decirle al browser que formato de video es. De otra manera este se descargaria algo del video para ver si puede reproducirlo (lo que tomaria tiempo y banda ancha) ejemplo: type='video/mp4;codecs="avc1.42E01E, mp4a.40.2"'

Existe otro atributo llamado codecs es utilizado para codificar el video y este va incluido dentro de type entre comillas simples ejemplo :type='video/mp4;codecs="avc1.42E01E, mp4a.40.2"'. Si el browser no soporta el elementro video, Este mostrara cualquier cosa que este entre la entrade y salida de video.

Anadiendo audio a la pagina web

Usar un servicio de host
Hay muchos sitios que permiten subir audios, y proveen un reproductor el cual tu puedes colocar en tu pagina, tales como soundCloud.com y myspace.com
Usar flash
Hay muchos tipos de flash movies que permiten reproducir archivos mp3 a traves de simples botones que reproducen los sonidos para los usuario que pueden crear playlist, etc
Usar html5
HTML5 ha introducido nuevos elementos como el elemento <audio>. Los browsers que soportan este elemento provee sus propios controles.(cuando se habla de controles se refiere a el boton de play y todo lo que va junto con el)

Para anadir un audio al igual que el elementro video hay que usar el elemento audio. Audio tiene una gran variedad de atributos al igual que video como:

src atributo que especifica la direccion del archivo

controls atributo que indica si deberian de mostrarse el control player, si no se usa este atributo no se mostrara, Puedes tambien especificar tu propio control player con javascript

autoplay la precencia de este atributo indica que el audio deberia de reproducirce automticamente.(Esto es considerado buena practica dejar al usuario elegir si reproducirlo o no)

preload este atributo indica que el browser deberia hacer si la reproduccion no esta seteada como automatica. Este puede tener el mismo valor que se vio con anterioridad en el elemento video (none, auto y metadata)

loop indica que el audio debe reproducirde una y otra vez cada vez que temine.

Entre el elemento audio existe un elemento llamado source que se encarga de especificar mas de un formato de reproduccion de audio, dejando el primer souce como el principal, en caso de no poder reproducirce ese , cambia al siguiente. Este elemento tiene atributos de type y src, al tener src significa que no hay que especificar src en audio, ejemplo:

<audio controls preload="none" >
<source src="audio/test-audio.ogg" / >
<source src="audio/test-audio.mp3" / >
<p >This browser does not support our audio format.</p >
</audio >
